## [0.1.0] — MVP (в разработке)

Первая версия приложения. Базовый каркас и основная функциональность.

### Добавлено

#### Каркас и дизайн-система
- Проект на SwiftUI + SwiftData, минимальная версия iOS 18
- Структура папок по слоям MVVM (App / Models / Services / ViewModels / Views / DesignSystem / Extensions)
- Дизайн-система: `AppColors` (семантические цвета — доход/долг/предупреждение), `AppFonts` (типографика с моноширинными цифрами для денег), `CardPalette` (палитра цветов карт)
- Главный `TabView` с 5 вкладками: Главная, Карты, Добавить, Калькулятор, Настройки
- Русская локаль и форматирование валют (`Decimal+Currency`)
- Расширение `Color+Hex` для работы с цветами карт

#### Карты
- Модели `Card` (debit/credit) с полным набором полей кредитки
- Создание, редактирование, удаление карт
- Список карт с цветными «пластиковыми» строками
- Детальный экран карты с градиентным дизайном
- Запрет смены типа карты при редактировании (защита от рассинхрона данных)

#### Категории
- Модель `Category` с иконками SF Symbols и цветами
- 13 предустановленных категорий при первом запуске (`CategoryProvider`)
- Экран управления категориями + создание своих

#### Транзакции
- Модель `Transaction` с типами: расход, доход, погашение, переводы
- Форма добавления операции с фильтрацией карт по типу операции
- `TransactionService` — единая точка обновления балансов карт
- Двойная запись для погашений (парные связанные транзакции через `linkedTransaction`)
- История операций на экране карты
- **Мягкое удаление (soft delete)** — отмена транзакции с откатом баланса и возможностью восстановления; полное удаление через свайп

#### Грейс-периоды и выписки
- Модель `Statement` (выписка как снэпшот расчётного периода)
- `DateHelper` — безопасная работа с календарём (защита от «31 февраля», часовых поясов)
- `StatementGenerator` — идемпотентная автогенерация выписок при запуске
- `StatementCalculator` — расчёт срочности (overdue/burning/soon/calm/paid), остатка к оплате, прогресса
- Визуальный блок «Активная выписка» с цветовой индикацией срочности и прогресс-баром
- История выписок на экране карты
- Правильное склонение «день/дня/дней»

#### Связь платежей и выписок
- `PaymentDistributor` — распределение платежа по выпискам карты (горящие первыми)
- Платёж автоматически уменьшает `paidAmount` выписок и закрывает их
- Откат распределения при отмене/удалении транзакции погашения

#### Умный калькулятор погашения
- `PaymentSuggester` — жадный алгоритм распределения свободных денег между кредитками
- Приоритизация выписок: просроченные → горящие → скоро → спокойные
- `CalculatorViewModel` + экран с вводом суммы, выбором карты-источника
- Карточки рекомендаций с обоснованием срочности и визуализацией «карта → карта»
- Сводка: сколько распределено, сколько выписок закроется
- Применение всего плана одной кнопкой

#### Регулярные расходы (резерв на жизнь)
- Модель `RegularExpense` с периодичностью (ежемесячно/еженедельно/ежедневно)
- `RegularExpenseCalculator` — расчёт резерва на 30 дней вперёд с учётом дат списания
- Экран управления регулярными расходами в Настройках
- Интеграция с калькулятором: из введённой суммы вычитается резерв, к погашению предлагается безопасный остаток

#### UX-улучшения
- Закрытие клавиатуры свайпом по форме (`scrollDismissesKeyboard`)
- Тосты-подтверждения при сохранении операций и применении платежей
- Glass-эффект (`.ultraThinMaterial`) на нижней панели с кнопками действий
- Пустые состояния (`ContentUnavailableView`) для всех списков
- Закреплённые кнопки действий внизу экрана вместо кнопок внутри формы

### Технические решения
- `Decimal` вместо `Double` для всех денежных сумм
- Сумма транзакции всегда положительная, направление — через `kind`
- Производные значения (срочность, дни до дедлайна) вычисляются на лету, не хранятся в БД
- Бизнес-логика изолирована в сервисах, View остаются «тонкими»

### Известные ограничения
- Нет production-миграций SwiftData (при изменении схемы во время разработки требуется переустановка)
- Дашборд («Главная») пока заглушка
- Только русский язык и рубли
- Нет импорта банковских выписок (только ручной ввод)
