# 💳 SqwizzeyFlow

> Нативное iOS-приложение для умного контроля личных финансов с акцентом на управление кредитными и дебетовыми картами, отслеживание грейс-периодов и оптимальное погашение долгов.

## 📖 О проекте

SqwizzeyFlow помогает не переплачивать банкам по кредитным картам. Приложение отслеживает льготные (грейс) периоды по каждой карте, понимает сколько и когда нужно внести, и **умно подсказывает**, с какой дебетовой карты на какую кредитку перевести деньги, чтобы закрыть долг без процентов — с учётом обязательных расходов на жизнь.

Все данные хранятся **локально на устройстве** (SwiftData). Никаких подключений к банкам, серверам или Apple Wallet — только ручной ввод. Это обеспечивает полную приватность.

## ✨ Ключевые возможности

### Реализовано (MVP)
- **Учёт карт** — добавление дебетовых и кредитных карт с настройкой лимита, долга, грейс-периода, дня выписки и дня платежа
- **Транзакции** — расходы, доходы и погашения с двойной записью (парные транзакции)
- **Категории расходов** — 13 предустановленных + создание своих с иконками и цветами
- **Грейс-периоды** — автоматическая генерация банковских выписок с расчётом дедлайнов и срочности
- **Умный калькулятор погашения** — распределяет свободные деньги между кредитками по приоритету (горящие выписки первыми)
- **Регулярные расходы** — учёт обязательных трат (аренда, продукты, подписки) для расчёта безопасной суммы к погашению
- **Мягкое удаление** — отмена транзакций с возможностью восстановления (soft delete)

### В планах
- Умные обоснования рекомендаций (расчёт переплаты при пропуске платежа)
- Дашборд со сводкой по всем картам
- Импорт банковских выписок с автокатегоризацией трат
- Локализация (английский язык)
- Push-уведомления о приближении дедлайнов
- Виджет на главный экран
- Face ID / Touch ID для входа

## 🏗 Архитектура

Проект построен по паттерну **MVVM** с выделенным слоем сервисов для бизнес-логики:

```
Models      →  @Model классы SwiftData (Card, Transaction, Category, Statement, RegularExpense)
Services    →  Чистая бизнес-логика (генерация выписок, расчёты, распределение платежей)
ViewModels  →  @Observable классы состояния экранов
Views       →  SwiftUI-экраны и компоненты
DesignSystem→  Цвета, шрифты, палитры — единая точка правды
Extensions  →  Хелперы (форматирование валют, hex-цвета, работа с клавиатурой)
```

## 🗃 Модель данных

```
Card (карта)
├── type: debit | credit
├── [debit]  balance
├── [credit] creditLimit, currentDebt, statementDay,
│             paymentDueDay, gracePeriodDays, interestRate
├── transactions: [Transaction]   (каскадное удаление)
└── statements: [Statement]       (каскадное удаление)

Transaction (операция)
├── amount (всегда положительное), date, kind, note
├── kind: expense | income | payment | transferOut | transferIn
├── isCancelled, cancelledAt       (мягкое удаление)
├── card, category
└── linkedTransaction              (парные переводы)

Statement (выписка)
├── periodStart, periodEnd, paymentDueDate
├── amountDue, minimumPayment, paidAmount
└── isClosed

Category (категория)
├── name, sfSymbol, colorHex, monthlyBudget?
└── transactions: [Transaction]

RegularExpense (регулярный расход)
├── name, amount, frequency, dayOfMonth
├── frequency: monthly | weekly | daily
└── sfSymbol, colorHex, isActive
```

### Ключевые принципы
- **`Decimal`, не `Double`** — для всех денежных сумм (точность копеек)
- **Сумма транзакции всегда положительная** — направление задаётся через `kind`
- **Двойная запись** — погашение создаёт две связанные транзакции (списание с дебетовки + погашение кредитки)
- **Производные значения не хранятся** — срочность выписки, дни до дедлайна и т.п. вычисляются на лету (зависят от текущей даты)

## 🧮 Как работает логика

### Грейс-периоды и выписки
Расчётный период (~30 дней) заканчивается в **день выписки** (`statementDay`). С этого момента начинается платёжный период до **дня платежа** (`paymentDueDay`). Если внести сумму выписки до дедлайна — процентов нет.

`StatementGenerator` при каждом запуске приложения проверяет каждую кредитку и создаёт недостающие выписки (идемпотентно — без дублей). Сумма выписки = все траты по карте за расчётный период.

`StatementCalculator` определяет срочность выписки:
- `overdue` — просрочена
- `burning` — ≤ 3 дней до дедлайна 🔥
- `soon` — ≤ 7 дней ⏰
- `calm` — > 7 дней
- `paid` — полностью оплачена ✅

### Умный калькулятор погашения
`PaymentSuggester` использует **жадный алгоритм**:
1. Собирает все активные выписки со всех кредиток
2. Сортирует по приоритету: `overdue → burning → soon → calm`, при равенстве — по убыванию суммы
3. Распределяет доступные деньги по очереди — сначала спасает то, что горит
4. Возвращает план: «с какой карты → на какую → сколько → почему»

### Резерв на жизнь
`RegularExpenseCalculator` считает сколько денег уйдёт на обязательные расходы за 30 дней вперёд (с учётом периодичности и дат списания). Калькулятор вычитает этот резерв из введённой суммы — к погашению предлагается только **безопасный остаток**.

## 🚀 Запуск проекта

### Требования
- macOS Sequoia 15.0+
- Xcode 16+
- Apple ID (для запуска на устройстве/симуляторе)

### Шаги
1. Клонировать репозиторий:
   ```bash
   git clone https://github.com/fairdenn/SqwizzeyFlow-for-iOS.git
   ```
2. Открыть `.xcodeproj` в Xcode
3. Выбрать команду разработки в **Signing & Capabilities** (свой Apple ID)
4. Выбрать симулятор (например, iPhone 16) или подключённое устройство
5. Запустить: **⌘R**

> ⚠️ При изменении моделей SwiftData во время разработки нужно удалить приложение из симулятора (зажать иконку → Удалить), чтобы пересоздать базу данных с чистой схемой. Production-миграции будут добавлены перед релизом.

## 🤝 Разработка

Проект разрабатывается соло, пошагово, в режиме обучения iOS-разработке. Каждый крупный модуль строится на предыдущем — см. [CHANGELOG.md](CHANGELOG.md) для истории.

### Соглашения
- Цвета — только через `AppColors`, шрифты — через `AppFonts`
- Денежные суммы — `Decimal`, форматирование — `.formattedRUB()`
- Строки писать через интерполяцию `\(value)`, а не конкатенацию — для будущей локализации
- Бизнес-логику выносить в `Services`, держать View «тонкими»
