# Sqwizzey Flow

Нативное iOS-приложение для умного контроля личных финансов. Акцент на кредитные карты — отслеживание грейс-периодов, расчёт дедлайнов и оптимальное распределение платежей.

---

## О проекте

Sqwizzey Flow помогает не переплачивать банкам. Приложение отслеживает льготные периоды по каждой карте, понимает сколько и когда нужно внести, и **умно подсказывает** с какой дебетовой карты на какую кредитку перевести деньги — с учётом обязательных расходов на жизнь.

Все данные хранятся **локально на устройстве** (SwiftData). Никаких подключений к банкам, серверам или Apple Wallet — только ручной ввод.

---

## Возможности

**Реализовано (MVP)**

- Учёт карт — дебетовые и кредитные с лимитом, долгом, грейс-периодом, днём выписки и днём платежа
- Транзакции — расходы, доходы и погашения с двойной записью (парные транзакции)
- Категории — 13 предустановленных + создание своих с иконками и цветами
- Грейс-периоды — автоматическая генерация выписок с расчётом дедлайнов и срочности
- Умный калькулятор — распределяет деньги между кредитками по приоритету (горящие выписки первыми)
- Регулярные расходы — учёт обязательных трат для расчёта безопасной суммы к погашению
- Мягкое удаление — отмена транзакций с возможностью восстановления

**В планах**

- Дашборд со сводкой по всем картам
- Умные обоснования рекомендаций (расчёт переплаты при пропуске платежа)
- Импорт банковских выписок с автокатегоризацией
- Push-уведомления о приближении дедлайнов
- Виджет на главный экран
- Face ID / Touch ID
- Локализация (English)

## Модель данных

```
Card
├── type: debit | credit
├── [credit] creditLimit, currentDebt, gracePeriodDays,
│            statementDay, paymentDueDay, interestRate
├── transactions: [Transaction]   (каскадное удаление)
└── statements:  [Statement]      (каскадное удаление)

Transaction
├── amount (всегда положительное), date, kind, note
├── kind: expense | income | payment | transferOut | transferIn
├── isCancelled, cancelledAt
└── linkedTransaction   (парные переводы)

Statement
├── periodStart, periodEnd, paymentDueDate
├── amountDue, minimumPayment, paidAmount
└── isClosed

RegularExpense
├── name, amount, frequency: monthly | weekly | daily
└── dayOfMonth, isActive
```

**Принципы:**
- `Decimal`, не `Double` — для всех денежных сумм
- Сумма транзакции всегда положительная, направление — через `kind`
- Двойная запись: погашение = две связанные транзакции
- Производные значения не хранятся — вычисляются на лету

## Как работает логика

### Грейс-периоды

`StatementGenerator` при запуске создаёт недостающие выписки для каждой кредитки (идемпотентно, без дублей). Сумма выписки = траты за расчётный период.

`StatementCalculator` определяет срочность:

| Статус | Условие |
|---|---|
| `overdue` | просрочена |
| `burning` | до 3 дней до дедлайна |
| `soon` | до 7 дней |
| `calm` | более 7 дней |
| `paid` | полностью оплачена |

### Умный калькулятор погашения

`PaymentSuggester` — жадный алгоритм:

1. Собирает все активные выписки
2. Сортирует: `overdue → burning → soon → calm`, при равенстве — по убыванию суммы
3. Распределяет деньги по очереди — сначала спасает то, что горит
4. Возвращает план: «с какой карты → на какую → сколько → почему»

### Резерв на жизнь

`RegularExpenseCalculator` считает сколько уйдёт на обязательные расходы за 30 дней вперёд. Калькулятор вычитает резерв — к погашению предлагается только безопасный остаток.

## Запуск

Требования: macOS Sequoia 15+, Xcode 16+

```bash
git clone https://github.com/fairdenn/SqwizzeyFlow-for-iOS.git
```

Открыть `.xcodeproj`, выбрать Apple ID в Signing & Capabilities, запустить (Cmd+R).

> При изменении моделей SwiftData — удалить приложение из симулятора для пересоздания базы.
